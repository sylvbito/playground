Add SPA-like page transitions to any multi-page site

Cross-document view transitions work in every modern browser. One CSS rule, zero JavaScript, and your concept sites feel native.

Three transition patterns

Each pattern is a few lines of CSS. Copy, paste, ship.

Crossfade

The default — enabled by a single rule. Old page fades out, new page fades in. Works everywhere.

Demo →
CSS 
@view-transition {
  navigation: auto;
}

Slide

Pages slide in from the right, creating forward/back navigation. Perfect for multi-step flows and galleries.

Demo →
CSS 
::view-transition-old(root) {
  animation: 0.4s slide-to-left both;
}
::view-transition-new(root) {
  animation: 0.4s slide-from-right both;
}

Shared Element Morph

Elements with the same view-transition-name on both pages animate between their positions. Hero images, titles, avatars — they morph seamlessly.

CSS 
/* On BOTH pages, give elements the same name */
.hero-title { view-transition-name: hero-title; }
.hero-image { view-transition-name: hero-image; }

One-liner starter

Add this to any page and all same-origin navigation gets a crossfade. That's it.

Add to <head> 
<!-- In <head> -->
<meta name="view-transition" content="same-origin">
<style>
@view-transition { navigation: auto; }
</style>
