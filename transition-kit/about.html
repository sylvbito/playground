How it works

🎯 One rule enables it

Add @view-transition { navigation: auto } to your CSS. That's it. Every same-origin navigation now has a smooth crossfade. Zero JavaScript.

🎨 Customise the animation

Override ::view-transition-old(root) and ::view-transition-new(root) with your own keyframes. Slide, scale, rotate — whatever fits.

🔄 Shared elements morph

Give elements on both pages the same view-transition-name. The browser animates them between their positions automatically.

🛡️ Progressive enhancement

Browsers without support navigate normally — no breakage. Your site works everywhere, looks better where supported.

Add it to your site in 3 steps

1

Add the meta tag + CSS rule

Drop <meta name="view-transition" content="same-origin"> in your <head> and add @view-transition { navigation: auto } to your stylesheet. Every page gets crossfades.

2

Customise per page

Override the animation on specific pages. Use different transitions for forward vs. back navigation, or different page types (products slide, blog crossfades).

3

Add shared-element morphs

Identify elements that appear on multiple pages (logos, hero images, titles). Give them the same view-transition-name. The browser handles the rest.
