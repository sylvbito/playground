Use case 1

Brand-driven UI panel

One --brand color drives the entire panel. Background, border, and hover states are derived with relative colors. Text and buttons use contrast-color() to stay readable automatically — no hand-picked text colors.

:root { --brand: oklch(0.6 0.22 260); /* Derived with relative colors */ --brand-bg: oklch(from var(--brand) calc(l + 0.32) calc(c * 0.15) h); --brand-border: oklch(from var(--brand) calc(l + 0.18) calc(c * 0.35) h); --brand-hover: oklch(from var(--brand) calc(l - 0.06) calc(c * 1.05) h); /* contrast-color() picks readable text automatically */ --brand-text: contrast-color(var(--brand-bg)); --brand-btn-text: contrast-color(var(--brand)); }
Use case 2

Auto-adaptive badge system

Four severity colors. Each badge derives its background from the severity via relative color (muted, tinted), and its text from that background via contrast-color(). Change any severity — everything stays readable.

.badge { background: oklch(from var(--severity) calc(l + 0.35) calc(c * 0.2) h); color: contrast-color(background-color); border: 1px solid oklch(from var(--severity) calc(l + 0.25) calc(c * 0.35) h); } .badge.success { --severity: oklch(0.62 0.19 160); } .badge.warning { --severity: oklch(0.72 0.17 85); } .badge.danger { --severity: oklch(0.55 0.22 20); } .badge.info { --severity: oklch(0.6 0.2 260); }
Use case 3

Generated palette — 5 colors, auto text

From one base color, five variants are derived using oklch(from ...). Each swatch uses contrast-color() on its own background — no manual text color assignment.

/* Five variants from one base */ .swatch-1 { background: var(--base); } .swatch-2 { background: oklch(from var(--base) calc(l - 0.15) c h); } .swatch-3 { background: oklch(from var(--base) calc(l + 0.28) calc(c * 0.25) h); } .swatch-4 { background: oklch(from var(--base) calc(l + 0.38) calc(c * 0.10) h); } .swatch-5 { background: oklch(from var(--base) calc(l - 0.08) calc(c * 1.15) calc(h + 180); } /* Every swatch auto-contrasts its own text */ .palette-swatch { color: contrast-color(background-color); }
Use case 4

Dynamic selection cards with auto-contrast

Three cards, each tinted from the brand color at different intensities using relative colors. contrast-color() handles the text. Click a card — its background deepens and text auto-flips.

/* Selected card uses a darker variant */ .option-card { --card-bg: oklch(from var(--brand) calc(l + var(--lift, 0.38)) calc(c * 0.12) h); background: var(--card-bg); color: contrast-color(var(--card-bg)); } .option-card.selected { background: oklch(from var(--brand) calc(l + 0.02) calc(c * 0.7) h); color: contrast-color(background-color); border-color: oklch(from var(--brand) l calc(c * 0.85) h); }
Use case 5

Multi-stop gradient card with auto-text

A gradient built from relative color stops. The card uses contrast-color() on a custom property set to the midpoint so text stays readable even as the gradient changes.

This text stays readable because contrast-color() checks the midpoint.
.gradient-card { /* Three stops derived from one brand color */ --grad-mid: oklch(from var(--brand) calc(l + 0.15) calc(c * 0.5) h); --grad-light: oklch(from var(--brand) calc(l + 0.36) calc(c * 0.15) h); --grad-dark: oklch(from var(--brand) calc(l - 0.04) calc(c * 0.7) h); background: linear-gradient(135deg, var(--grad-light), var(--grad-mid), var(--grad-dark)); color: contrast-color(var(--grad-mid)); }
