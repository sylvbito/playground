1. Anchor-Positioned Tooltip

Hover-triggered tooltip using CSS Anchor Positioning. Automatically flips when near viewport edges. Zero JavaScript.

anchor-nameposition-areaposition-try-fallbacks
Hover the button
⌘S to save
This cannot be undone
/* Anchor declared on the trigger */
.tooltip-btn { anchor-name: --tt-anchor; }

/* Tooltip positioned relative to anchor */
.tooltip-content {
  position: absolute;
  position-anchor: --tt-anchor;
  position-area: top;
  position-try-fallbacks: flip-block;
  opacity: 0;
  transition: opacity 0.15s, transform 0.15s;
}

.tooltip-btn:hover + .tooltip-content {
  opacity: 1;
  transform: translateY(0);
}

2. Popover Dropdown Menu

Click-triggered dropdown using the Popover API and Anchor Positioning. Light-dismiss, focus management, and keyboard navigation built in.

popoverpopovertargetanchor-name@starting-style
Click the button 
<!-- Trigger button -->
<button popovertarget="menu" style="anchor-name:--dd">Actions ▾</button>

<!-- Popover menu -->
<div id="menu" popover="auto" style="position-anchor:--dd">
  <button>Edit</button>
  <button>Delete</button>
</div>

/* CSS */
.dropdown-menu {
  position: absolute;
  position-area: bottom;
  position-try-fallbacks: flip-block;
  opacity: 0;
  transition: opacity 0.15s, transform 0.15s;
  transition-behavior: allow-discrete;
}

.dropdown-menu:popover-open { opacity: 1; }

@starting-style {
  .dropdown-menu:popover-open { opacity: 0; }
}

3. Animated Dialog

Modal dialog with backdrop blur and scale-up entry animation. Uses Popover API for top-layer, light-dismiss, and focus trapping. @starting-style handles the entry transition.

popover@starting-style::backdroptransition-behavior
Click to open

Confirm action

Are you sure you want to proceed? This dialog is built entirely with CSS. No JavaScript required.

<!-- Trigger -->
<button popovertarget="dlg">Open</button>

<!-- Dialog overlay (popover) -->
<div id="dlg" popover="auto" class="overlay">
  <div class="box">
    <h3>Title</h3>
    <button popovertarget="dlg" popoveraction="hide">Close</button>
  </div>
</div>

/* Entry animation */
.overlay:popover-open .box {
  opacity: 1;
  transform: scale(1);
  transition: opacity 0.25s, transform 0.25s;
  transition-behavior: allow-discrete;
}
@starting-style {
  .overlay:popover-open .box {
    opacity: 0;
    transform: scale(0.95);
  }
}

4. Custom Select (appearance: base-select)

Fully styleable native <select> — no JavaScript library needed. Customise the button, dropdown, options, and checkmarks. Gracefully degrades in non-Chromium browsers.

appearance::picker(select)::checkmark
Native select, fully styled 
select {
  appearance: base-select;
  padding: 0.5rem 2rem 0.5rem 0.75rem;
  border-radius: 6px;
  border: 1px solid #333;
  background: #222;
  color: #fff;
  font-family: system-ui;
}

select::picker(select) {
  background: #2a2a2a;
  border-radius: 10px;
  box-shadow: 0 8px 32px rgba(0,0,0,0.5);
  padding: 0.25rem;
}

select option::checkmark { color: dodgerblue; }

/* Graceful fallback */
@supports not (appearance: base-select) {
  select { appearance: auto; }
}

5. Cascade-Layered Component

Use CSS @layer to control cascade priority at the component level. Base styles in one layer, overrides in another — specificity doesn't matter, layer order does.

@layer
Same specificity, different layers

Standard Card

Styled by the @layer components layer.

Special Card

Styled by @layer overrides which wins regardless of specificity.

/* components layer — base styles */
@layer components {
  .card {
    background: #222;
    padding: 1.25rem;
    border-radius: 10px;
  }
}

/* overrides layer — wins even with same specificity */
@layer overrides {
  .card.special {
    background: oklch(0.7 0.2 250 / 0.15);
  }
}

/* Layer order declared at top of stylesheet */
@layer components, overrides;
