Anchor-Positioned Tooltips

Hover any icon. Each tooltip is tethered via position-anchor with position-try-fallbacks for edge-aware repositioning. Zero JavaScript for positioning.

Glass Morphism Backdrop-filter blur creates depth separation. The browser composites this on the GPU — zero repaints.
position-area Declares where the tooltip sits relative to its anchor. top center → above, centered.
Perceptual Colour OKLCH gradients maintain uniform lightness across hue shifts. No muddy midpoints.
position-try-fallbacks When the tooltip overflows the viewport, the browser flips it to the opposite side. Automatic edge detection.
anchor-size() The tooltip width matches its anchor via anchor-size(width). No JavaScript width sync.
@layer Cascade All CSS is organised in explicit cascade layers — reset → theme → layout → components → utilities.

Edge-Aware Fallbacks

position-try-fallbacks lets you specify a chain of fallback positions. The browser picks the first that fits.

Click for details →
Try chain: right → left → bottom

Each fallback is evaluated in order. If the positioned element overflows in its current position, the browser tries the next one. No JavaScript viewport math.

Anchor Scope

The anchor-scope property limits which anchors are visible to which positioned elements — essential for nested components.

Click for details →
Without scope: All anchors on the page compete for positioned elements.

With scope: anchor-scope: --my-component isolates anchors to a subtree. Essential for design systems with nested anchor-positioned patterns.
