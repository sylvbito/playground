▸ Full-viewport displacement map approach

1A canvas generates a viewport-sized displacement map (at half resolution). The lens shape — a rounded rect with a sinusoidal gradient — is drawn at the current cursor position. Everywhere else is neutral (R=128, G=128) = no displacement.
2The SVG <filter> uses default bounds (no x/y/width/height) — the filter covers the full element. feImage loads the map at (0, 0, viewportW, viewportH).
3feDisplacementMap samples the map at each pixel. Neutral map pixels → no displacement (content renders normally). Map pixels at the lens position → displacement → refraction effect.
4Each frame: assign a fresh filter id (Safari cache workaround). The map is regenerated only when the lens moves >3 px — cached per quantized position.
Single <filter> with default bounds — no filter-region clipping
Fresh filter id every frame — defeats Safari's filter output cache
Full-viewport canvas map (½ res) — lens drawn at cursor position, neutral elsewhere
Map regenerated on >3 px movement — Uint32Array fill for fast neutral clearing
<filter id="glass-42"
  filterUnits="userSpaceOnUse"
  colorInterpolationFilters="sRGB">
  <!-- No x/y/w/h — defaults cover -10% to 120% -->
  <feImage href="full-map.png"
    x="0" y="0"
    width="{vw}" height="{vh}"
    result="map"/>
  <feDisplacementMap
    in="SourceGraphic"
    in2="map"
    scale="45"/>
</filter>
